# 게임용 네트워크 레지스트리 튜닝 도구 — 요구사항

## 1. 목적

- 게임용 네트워크 레지스트리 튜닝값을 **GUI 버튼으로 ON/OFF**
- **exe로 빌드**하여 더블클릭 실행

## 2. 대상 레지스트리 값

| 값 이름 | 적용값 | 경로 |
| --- | --- | --- |
| `TcpAckFrequency` | `1` | 선택한 어댑터 GUID 경로 |
| `TCPNoDelay` | `1` | 선택한 어댑터 GUID 경로 |
| `NetworkThrottlingIndex` | `0xFFFFFFFF` | Multimedia SystemProfile |

### 2.1 값 상세 설명

#### TcpAckFrequency

- **경로**: `HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\Interfaces\{어댑터 GUID}`
- **타입**: `REG_DWORD`
- **의미**: TCP ACK를 몇 개 패킷마다 보낼지 결정. 기본은 지연 ACK(여러 패킷 모아 ACK)
- **기본값**: 값 없음 (= 기본 동작 `2`, 지연 ACK 활성)
- **튜닝값**: `1` → 매 패킷마다 즉시 ACK (지연 ACK 사실상 비활성)
- **효과**: 수신측 ACK 지연 제거 → 왕복 응답 지터 감소. 단 ACK 패킷 수 증가
- **주의**: 어댑터별 GUID 경로라 어댑터마다 따로 설정해야 함

#### TCPNoDelay (Nagle 비활성)

- **경로**: 위 `TcpAckFrequency`와 동일한 어댑터 인터페이스 경로
- **타입**: `REG_DWORD`
- **의미**: Nagle 알고리즘(작은 패킷을 모아 전송) 사용 여부
- **기본값**: 값 없음 (= Nagle 활성)
- **튜닝값**: `1` → Nagle 비활성, 작은 패킷도 즉시 전송
- **효과**: 송신측 패킷 묶음 지연 제거. **ping에 그나마 직접 닿는 항목** — 특정 게임에서 수 ms 단축
- **주의**: `TcpAckFrequency`와 **짝으로 함께 적용**해야 의도한 효과. 대역폭 효율은 약간 저하

#### NetworkThrottlingIndex

- **경로**: `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Multimedia\SystemProfile`
- **타입**: `REG_DWORD`
- **의미**: 멀티미디어 재생 중 비-멀티미디어 네트워크 패킷 처리율 제한(기본 10패킷/ms)
- **기본값**: `10` (= throttling 활성)
- **튜닝값**: `0xFFFFFFFF` (= `4294967295`) → throttling 완전 비활성
- **효과**: **멀티미디어 앱 실행 + 고pps 부하** 상황에서 패킷 처리 제한 해제 → 지터 완화
- **주의**: **ping 단축용 아님**. 시스템 전역 값(어댑터 무관). 변경 후 반영에 재부팅 권장

> 참고: 세 값 모두 **물리적 RTT(거리·ISP·서버 위치)는 줄이지 못함**. 로컬 처리 지연/지터에만 영향.

## 3. GUI 기능

- 네트워크 어댑터 **자동 감지 + 드롭다운 선택** (GUID 수동 입력 불필요)
- **ON 버튼 / OFF 버튼 / 현재 상태 표시 / 로그창**
- ON/OFF 상태에 따라 버튼 **자동 활성·비활성**

## 4. 동작 정의

### ON 동작

1. 현재값을 **디스크에 백업** (없던 값은 `"없음"`으로 기록)
2. 튜닝값 적용
3. 어댑터 재시작

- 이미 ON이면 **거부** (중복 적용 차단)

### OFF 동작

1. 백업 기준 **정확히 원상복구** (원래 없던 값은 삭제)
2. 백업 삭제
3. 어댑터 재시작

- 백업 없으면 **거부**

## 5. 기술 / 실행 요건

- **Python + tkinter** (표준 라이브러리만, 외부 의존성 없음)
- **관리자 권한 필요** (HKLM 쓰기 + 어댑터 재시작), 미보유 시 **UAC 자동 요청**
- 레지스트리 작업은 **별도 스레드** (어댑터 재시작 중 UI 멈춤 방지)
- **PyInstaller 빌드**: `--onefile --windowed --uac-admin`

## 6. 안정성 / 엣지케이스 (필수)

- **어댑터 재시작 실패 시 자동 롤백** — ON 중 disable/enable 실패 시 레지스트리를 백업으로 복구하고 어댑터 재활성 보장 (레지스트리만 바뀌고 어댑터가 죽는 상태 방지)
- **재시작 vs 재부팅 선택** — `TcpAckFrequency`/`TCPNoDelay`는 어댑터 재시작 또는 재부팅으로 반영. "재시작 대신 재부팅 안내" 옵션을 제공해 끊김 리스크를 사용자가 선택
- **백업 파일 고정 경로 저장** — `--onefile` exe는 임시폴더에 풀리므로 백업은 exe 옆이 아닌 `%APPDATA%` 등 고정 경로에 저장
- **백업 손상/수동삭제 대비** — 백업 JSON이 손상·삭제된 경우의 OFF 동작 정의 필요 (단순 거부 외 처리)
- **어댑터 식별자 동시 저장** — 드라이버 재설치 시 GUID 변동 대비, 백업에 어댑터 이름/MAC을 함께 저장해 OFF 시 매칭 검증

## 7. 상태 판정 정확성

- **부분 적용 상태 표시** — 3개 값 중 일부만 적용된 경우 ON/OFF 외 **"불일치"** 상태로 표시 (외부 변경 감지)
- **실제 레지스트리 값과 대조** — 상태를 백업파일 유무로만 판단하지 않고 현재 레지스트리 값과 직접 비교

## 8. UX / 운영

- **현재 적용값 표시** — 상태창에 각 레지스트리 값의 현재값 표시 (신뢰도 향상)
- **기본값 강제 초기화 버튼** — 백업이 없어도 알려진 기본값(throttling=`10` 또는 삭제)으로 되돌리는 비상 복구
- **로그 파일 저장** — UI 로그창 외 파일로도 저장하여 문제 추적
- **UAC 거부 시 정상 종료** — 관리자 권한 재요청 루프 방지

## 9. 제약 / 인지사항

- 어댑터 재시작 시 **수 초간 네트워크 끊김**
- 효과는 **로컬 처리 지연/지터 한정** — 물리적 RTT(거리·ISP·서버 위치)는 줄지 않음
- `NetworkThrottlingIndex`는 ping 단축용 아님 → **멀티미디어 앱 + 고pps 부하** 상황의 지터 완화용
- ping에 그나마 직접 닿는 건 **Nagle 끄기**뿐이며, 특정 게임에서 수 ms 수준

## 10. 문서 / 배포

- PyInstaller `--onefile` exe는 **백신 오탐 가능** → 필요시 `--onedir` 또는 **코드 서명**
- **Defender 오탐 대응 구체화** — 빌드 재현성 확보, 오탐 제출(false positive submission) 경로 안내
- **테스트 체크리스트 작성**
  - ON → 재부팅 → 값 유지 확인
  - OFF → 원상복구(없던 값 삭제 포함) 확인
  - 백업 삭제 후 OFF 거부 동작 확인
  - 어댑터 재시작 실패 시 롤백 동작 확인
  - 이미 ON 상태에서 ON 재실행 거부 확인
